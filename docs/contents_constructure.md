# 기본 컨텐츠 구조 및 구성
## 컨텐츠 구성
    1. External Music Pool(EMP)
        - 각 스트리밍 플팻폼에서 추천하는 플레이리스트, 매거진 트랙들을 크롤링하여 구성
        - 사용자가 회원가입할때 기본적으로 제공하는 각종 플레이리스트, 트랙들
        - 각 스트리밍 플랫폼의 공유 플레이리스트 불러오기 한 내역들
        - 등등 풀을 확장할 수 있는 각종 수단들을 최대한 활용하여 구성된 풀로 스케줄링을 이용하여 영속성을 확보함
    2. Model Recomendation Tracks(MRT)
        - 추천앨범, 추천플레이리스트, 추천트랙 등으로 구성됨
        - 개인별로 제공 되는 영역으로 PGT를 기반으로 사용자의 취향을 파악하고 유사한 곡들을 추천하는 영역
        - EMP와 PGT를 기반으로 구성됨
        - 사용자의 요청과 주기적으로 업데이트됨
        - 스케줄과 관련된 전문 관리 어드민 페이지에서 관리됨
        - 기본값은 1주일에 2번 업데이트됨
        - 사용자의 선택에 의해서 PGT로 이동됨
    3. Personal Generated Tracks(PGT)
        - 사용자가 직접 생성하는 영역으로 다른 영역과 상호작용을 통해 사용자의 취향을 파악하고 유사한 곡들이 추천되어 사용자와 함께 만들어가는 영역
        - 외부 스트리밍 플렛폼에서 회원가입할때 자동으로 시스템에 임포트된 자신의 플레이리스트, 앨범, 아티스트, 좋아요한 트랙
        - 외부적으로 재생되는 용도의 트랙 모음은 아니지만 직접적으로 사용자의 취향을 나타내는 트랙들을 모아 놓은 Personal Core Tracks(PCT)이 있고 최초 외부 플랫폼 임포트 된값으로 생성된 이후 순수 사용자의 선택으로 PGT등 에서 선택됨
        - 어떠한 페이지 이든 사용자가 '내 취향이예요'를 누를경우 PGT에 추가됨과 동시에 PCT에도 추가됨
## Major Streaming Platform
    1. 회원가입시 자동으로 연동되서 PGT가 구성되는 플렛폼 및 EMP에 가져오는 플렛폼
    - Tidal
    - Spotify
    2. 회원가입시 자동으로 연동되서 PGT가 구성되지 않지만 EMP에서 가져오는 플렛폼
    - Apple Music
    - Youtube Music
    - FLO
    - Genie
    - Bugs
    - Melon
    - Amazon Music
    - Deezer

## 회원가입 및 온보딩 프로세스
    1. 회원가입
        - 이메일, 패스워드, 닉네임, 구독중인 스트리밍 플랫폼 으로 회원가입
    2. 스트리밍 플렛폼 연동
        - Tidal
        - Spotify
    3. 스트리밍 플랫폼 인증(OAuth 2.0)
    4. 관련 권한 획득
    5. 플레이리스트, 좋아요한 곡, 앨범, 아티스트 등 개인 데이터 임포트
    6. 임포트된 데이터를 기반으로 컨텐츠 구조에 명시된 PGT 구성
    7. 추천 모델에 의해서 자동으로 MRT 컨텐츠 생성 후 제공
